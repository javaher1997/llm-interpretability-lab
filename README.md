# Layer-wise and Token-wise Analysis of Language Model (GPT-2) Representations

This repository provides tools for in-depth analysis of the internal representations (hidden states) generated by the GPT-2 language model. It allows for both layer-wise examination of token evolution and token-focused analysis within specific layers.

## Getting Started: Running the Analysis

This project involves two main stages: extracting hidden states and then analyzing them.

### Stage 1: Extracting Hidden States (`extract_hidden_states.py`)

This script uses the `transformers` library to load a GPT-2 model and tokenizer, processes a list of prompts defined in `config.py`, and saves the extracted hidden states and token information.

*   **Prerequisites:**
    *   Python 3.x
    *   PyTorch (`torch`)
    *   Hugging Face `transformers` library
    *   The `config.py` file must be present in the same directory or accessible by the script.

*   **Configuration (`config.py`):**
    Before running, ensure `config.py` is set up correctly:
    *   `PROMPTS`: A list of input strings you want to process.
    *   `MODEL_NAME`: The specific GPT-2 variant to use (e.g., "gpt2", "gpt2-medium").
    *   `OUTPUT_DIR`: Directory where the hidden states file will be saved (defaults to `data/hidden_states`).

*   **How to Run:**
    The script is designed to be run modularly. Ensure you are in the directory containing `extract_hidden_states.py` and `config.py`.
```bash
python -m extract_hidden_states
```
*(Note: If your project structure requires a different way to import/run, adjust the command accordingly. For example, if `extract_hidden_states.py` is inside a `scripts` folder, you might use `python -m scripts.extract_hidden_states` from the parent directory.)*
for example: 
```bash
llm-interpretability-lab> python -m  experiments.extract_hidden_states
```

Expected Output:A PyTorch file (default: data/hidden_states/gpt2_hidden_states.pt) will be created, containing a dictionary. Each key is a prompt, and the value is another dictionary holding:
input_ids: The token IDs for the prompt.
tokens: The string representation of the tokens.
hidden_states: A list of PyTorch tensors, where each tensor represents the hidden states for a specific layer (including the embedding layer).

**cross_prompt_analysis.py** (Cross-Prompt Token Comparison)
Purpose: Compares token representations between two different prompts within the same layer, or tracks a specific token’s representation across prompts and layers.
How to Run:
```bash
python cross_prompt_analysis.py
```
*(Make sure `PROMPT_PAIR` and `DATA_PATH` are correctly set in `config.py`. You might also need to specify `LAYER_INDEX` if the script is designed to compare within a specific layer.)*
Expected Output: Visualizations (e.g., similarity matrices between prompts, comparative plots of token embeddings) that highlight differences or similarities in how tokens are represented across different input contexts.

**layerwise_analysis.py** (Layer-wise Token Evolution)
Purpose: Analyzes the evolution of a specific, target token’s representation across all layers.
How to Run:
```bash
python -m layerwise_analysis
```
*(Make sure `TARGET_TOKEN` and `DATA_PATH` are correctly set in `config.py`.)*
Expected Output: Visualizations (e.g., plots using matplotlib) showing the target token’s representation changes across layers (e.g., norm, similarity).

**token_focus_analysis.py** (Token Comparison within a Layer)
Purpose: Compares the representations of all tokens within a single, specified layer.
How to Run:
```bash
python -m token_focus_analysis
```
*(Make sure `LAYER_INDEX` and `DATA_PATH` are correctly set in `config.py`.)*
Expected Output: Visualizations (e.g., heatmaps, similarity matrices) illustrating relationships between tokens in the chosen layer.
